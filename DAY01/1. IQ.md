#  30 Essential Linux Commands for DevOps Engineers

A curated list of Linux commands frequently used in DevOps for automation, CI/CD, troubleshooting, deployments, system administration, and networking.

---

## 1. **ls**

**List directory contents**
**Syntax:** `ls [options]`
**Example:** `ls -l`
**Use case:** View files with detailed permissions and ownership.

---

## 2. **cd**

**Change directory**
**Syntax:** `cd <directory>`
**Example:** `cd /etc`
**Use case:** Navigate file system paths during troubleshooting.

---

## 3. **pwd**

**Print current directory**
**Syntax:** `pwd`
**Example:** `pwd`
**Use case:** Confirm working directory in scripts or SSH sessions.

---

## 4. **mkdir**

**Create a directory**
**Syntax:** `mkdir <dir>`
**Example:** `mkdir logs`
**Use case:** Create folder structures during deployments.

---

## 5. **rm**

**Remove files or directories**
**Syntax:** `rm [options] <path>`
**Example:** `rm -rf /tmp/cache`
**Use case:** Clean temporary files or build artifacts.

---

## 6. **cp**

**Copy files or folders**
**Syntax:** `cp <src> <dest>`
**Example:** `cp config.yaml /backup/`
**Use case:** Backup config files before editing.

---

## 7. **mv**

**Move or rename files**
**Syntax:** `mv <src> <dest>`
**Example:** `mv app.log app.log.old`
**Use case:** Log rotations, renaming config files.

---

## 8. **cat**

**View file content**
**Syntax:** `cat <file>`
**Example:** `cat /etc/os-release`
**Use case:** Quick file inspection.

---

## 9. **tail**

**View last lines of file**
**Syntax:** `tail [-f] <file>`
**Example:** `tail -f /var/log/syslog`
**Use case:** Monitor logs in real time.

---

## 10. **head**

**View first lines of file**
**Syntax:** `head <file>`
**Example:** `head /etc/passwd`
**Use case:** Inspect beginnings of large files.

---

## 11. **grep**

**Search text patterns**
**Syntax:** `grep "<pattern>" <file>`
**Example:** `grep "error" app.log`
**Use case:** Log searching and debugging.

---

## 12. **find**

**Search files and directories**
**Syntax:** `find <path> -name "<pattern>"`
**Example:** `find /var -name "*.log"`
**Use case:** Locate configs, logs, or scripts.

---

## 13. **ps**

**View running processes**
**Syntax:** `ps aux`
**Example:** `ps aux | grep nginx`
**Use case:** Debug process issues or resource usage.

---

## 14. **top**

**Live view of system processes**
**Syntax:** `top`
**Use case:** Monitor CPU, memory usage in real time.

---

## 15. **htop**

**Interactive process viewer (improved top)**
**Syntax:** `htop`
**Use case:** Debug server performance (color UI).
*(Requires installation)*

---

## 16. **systemctl**

**Control systemd services**
**Syntax:** `systemctl <action> <service>`
**Example:** `systemctl restart nginx`
**Use case:** Start/stop/restart services.

---

## 17. **journalctl**

**View systemd logs**
**Syntax:** `journalctl -u <service>`
**Example:** `journalctl -u docker -f`
**Use case:** Troubleshoot service-level logs.

---

## 18. **service**

**Legacy service management**
**Syntax:** `service <service> <command>`
**Example:** `service ssh status`
**Use case:** Manage older system services.

---

## 19. **df**

**Disk usage overview**
**Syntax:** `df -h`
**Use case:** Check disk availability for deployments.

---

## 20. **du**

**Directory size usage**
**Syntax:** `du -sh <path>`
**Example:** `du -sh /var/log`
**Use case:** Find large directories using space.

---

## 21. **tar**

**Archive or extract files**
**Syntax:**

* Create: `tar -czf file.tar.gz dir/`
* Extract: `tar -xzf file.tar.gz`
  **Use case:** Packaging backups or artifacts.

---

## 22. **curl**

**Send HTTP requests**
**Syntax:** `curl <url>`
**Example:** `curl -I https://google.com`
**Use case:** Test HTTP endpoints during CI/CD.

---

## 23. **wget**

**Download files from the web**
**Syntax:** `wget <url>`
**Example:** `wget https://domain.com/app.tar.gz`
**Use case:** Download binaries or artifacts.

---

## 24. **ssh**

**Remote login to servers**
**Syntax:** `ssh user@host`
**Example:** `ssh ubuntu@52.43.12.11`
**Use case:** Access cloud VMs (AWS, Azure, GCP).

---

## 25. **scp**

**Secure copy between hosts**
**Syntax:** `scp <src> <user@host:/dest>`
**Example:** `scp build.zip ubuntu@server:/var/www/`
**Use case:** Transfer build packages to servers.

---

## 26. **chmod**

**Change file permissions**
**Syntax:** `chmod 755 <file>`
**Example:** `chmod +x deploy.sh`
**Use case:** Make scripts executable.

---

## 27. **chown**

**Change file ownership**
**Syntax:** `chown user:group <file>`
**Example:** `chown root:root /etc/hosts`
**Use case:** Fix permission issues.

---

## 28. **env**

**Show environment variables**
**Syntax:** `env`
**Example:** `env | grep PATH`
**Use case:** Debug CI/CD environment issues.

---

## 29. **export**

**Set environment variables**
**Syntax:** `export VAR=value`
**Example:** `export JAVA_HOME=/usr/lib/jvm/java-11`
**Use case:** Temporary environment configuration.

---

## 30. **crontab**

**Schedule recurring jobs**
**Syntax:** `crontab -e`
**Example:**

```
0 2 * * * /usr/bin/backup.sh
```

**Use case:** Automated backups, log cleanup, periodic tasks.

---
